# Sheet Schema — CozyCatKitchen

Single source of truth for the Google Sheets data model used by the Orders
web app. All backend code reads and writes columns by the header names defined
in `ORDER_COLS` / `CUSTOMER_COLS` in `backend/orders/Code.js` — **never by
position** — so reordering or inserting a column in the Sheet is safe.

## Sheets overview

| Sheet name | Purpose |
|---|---|
| Orders (active/default sheet) | One row per submitted bill |
| Customers | One row per unique customer, auto-upserted on each order |
| Users | Login credentials for app users |
| ErrorLog | Client-side errors reported by the web app |

---

## Orders

Managed by: `backend/orders/Code.js`

| Column | Header | Type | Notes |
|---|---|---|---|
| A | Bill No | String | Unique per order, e.g. `CCK-20240101-001` |
| B | Date | String | Display date, e.g. `01 Jan 2024` |
| C | Customer Name | String | Free text |
| D | Phone | String | |
| E | Email | String | |
| F | Address | String | May be multi-line |
| G | Items Summary | String | Semicolon-delimited, e.g. `Cat: Chicken Pâté x2 (₹320); Cat: Tuna Mousse x1 (₹160)` |
| H | Total Items | Number | Count of individual units ordered |
| I | Delivery Charges | Number | ₹ amount; 0 for pickup orders |
| J | Total Amount | Number | ₹ amount including delivery |
| K | Payment Status | Enum | `Pending` \| `Paid` \| `Refunded` \| `Failed` \| `Cancelled` |
| L | Dispatch Date | String | Optional; shown on the bill |
| M | Remarks | String | Optional; shown on the bill |
| N | Generated By | String | Username of the staff member who created the bill |
| O | Payment Proof | String | Google Drive URL; filled by the upload-proof flow |
| P | Fulfillment Status | Enum | `Packed` \| `Booked` \| `Picked Up` \| `Delivered`; blank until the order moves |
| Q | Tracking Link | String | Shipper tracking URL; filled when status reaches Booked or later |
| R | Map Link | String | Google Maps link for the delivery address |
| S | Delivery Type | Enum | `Delivery` \| `Pickup` |
| T | Share Token | String | UUID scoping the customer-facing tracking link to this order |

**Sheet setup helpers** (run once from the Apps Script editor):
- `setupPaymentStatusDropdown()` — adds a dropdown to the Payment Status column
- `setupFulfillmentDropdown()` — adds a dropdown to the Fulfillment Status column

Both helpers derive the column letter from the live header row, so they work
even if columns are reordered.

## Customers

Managed by: `backend/orders/Code.js` (`getCustomers`, `upsertCustomer`, `migrateCustomersFromOrders`)

| Column | Header | Type | Notes |
|---|---|---|---|
| A | Name | String | Matched case-insensitively on upsert |
| B | Phone | String | Overwritten with the most recent order's value |
| C | Address | String | Overwritten with the most recent order's value |
| D | Email | String | Overwritten with the most recent order's value |
| E | Notes | String | Manual notes; **never overwritten by code** |
| F | LastOrderDate | String | `dd/MM/yyyy`; updated on each new order |
| G | TotalOrders | Number | Incremented on each new order |

---

## Users

Managed by: `backend/orders/Code.js` (`setupUser`, `findUser_`, `usersSheet_`)

| Column | Header | Type | Notes |
|---|---|---|---|
| A | username | String | Case-insensitive match on login |
| B | salt | String | UUID, unique per user |
| C | passwordHash | String | `base64url(SHA-256(password + salt))` |
| D | role | String | `admin` \| `staff`; the app grants full access to `admin` |
| E | active | Boolean | Set to `FALSE` to disable without deleting the record |

---

## ErrorLog

Written by: `backend/orders/Code.js` (`logClientError_`)
Triggered by: frontend `?action=clientError` GET calls

| Column | Header | Type | Notes |
|---|---|---|---|
| A | Timestamp | Date | Apps Script `new Date()` at time of logging |
| B | User | String | Username from token if present; blank for logged-out errors |
| C | Message | String | Error message, truncated to 500 chars |
| D | Context | String | Page context, e.g. `track`, `orders`, truncated to 120 chars |
| E | Path | String | URL path that triggered the error, truncated to 200 chars |
| F | UserAgent | String | Browser user-agent string, truncated to 300 chars |

Capped at 500 entries; oldest rows are deleted automatically on each write.

## Adding a new column

1. Add the header string to `ORDER_COLS` (or `CUSTOMER_COLS`) in `backend/orders/Code.js`.
2. Add it to `ORDER_HEADERS` (or `CUSTOMER_HEADERS`) in the desired position.
3. Update `buildRow_()` callers in `doPost` if the new column is populated on order creation.
4. Add it to this document.
5. Deploy via `clasp push` — see `docs/CLASP_SETUP.md`.

Existing rows will have a blank cell for the new column until backfilled manually.
